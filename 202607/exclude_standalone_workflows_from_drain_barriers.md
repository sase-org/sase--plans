# Plan: Exclude Standalone Xprompt Workflows from Drain Barriers

## Goal

Correct the recently added chop drain barriers so `%w(runners=0)` appears only on prompts that launch ordinary root user
agents. Prompts that dispatch a standalone xprompt workflow, identified by an explicit `#!name` reference, must not
carry the barrier. Preserve barriers on the ordinary detached agents launched by those workflows and on other
non-workflow agent prompts.

This is a prompt-source and regression-test correction. Do not change runner-slot admission, directive parsing,
standalone workflow flattening, chop schedules, workflow structure, or launch APIs.

## Current-State Classification

The previous drain-barrier change covered both workflow wrappers and their detached leaf agents. The wrapper half is too
broad because `#!` prompts dispatch workflow execution rather than representing the root user-agent work that the
barrier is intended to gate.

The current launch surfaces divide into two classes:

1. **Standalone workflow wrapper prompts, which must not contain `%w(runners=0)`:**
   - All eight `agent:`-backed chops in `home/dot_config/sase/sase_athena.yml`:
     - `sase_toobig_split`
     - `sase_recent_bug_audit`
     - `sase_recent_improvement_audit`
     - `sase_refresh_docs`
     - `sase_core_refresh_docs`
     - `sase_github_refresh_docs`
     - `sase_nvim_refresh_docs`
     - `sase_telegram_refresh_docs`
   - `FIX_JUST_PROMPT` in `home/bin/executable_sase_chop_sase_fix_just`.

   Each of these prompts contains one explicit `#!sase/...` reference. Remove only the runner drain directive while
   retaining names, VCS references, chop groups, `%auto`, workflow arguments, and formatting.

2. **Ordinary root-agent prompts, which should retain exactly one `%w(runners=0)`:**
   - The GitHub Actions fixer prompt built by `home/bin/executable_sase_chop_gh_actions_fix`; it contains no standalone
     workflow reference and directly launches the fixer agent.
   - The detached split-file prompts generated by `xprompts/toobig_split.yml`.
   - The detached audit prompts generated by `xprompts/audit_recent_bugs.yml` and
     `xprompts/audit_recent_improvements.yml`.
   - The detached update and polish prompts generated by `xprompts/refresh_docs.yml`.

   The bundled workflow leaf prompts may contain ordinary inline xprompts such as `#split_file`, `#sase/docs`, or
   `#fork`; those are not standalone `#!` workflow references and do not fall under this correction.

## Implementation

### 1. Remove barriers from the Athena standalone-workflow wrappers

In the linked chezmoi repository, remove `%w(runners=0)` from all eight `agent:` values in
`home/dot_config/sase/sase_athena.yml`. Preserve the exact `#!` workflow invocation and all other directives and
arguments, including folded YAML formatting for the project-specific refresh-docs prompts.

Remove the barrier from `FIX_JUST_PROMPT` in `home/bin/executable_sase_chop_sase_fix_just`, leaving it as a named,
chop-tagged invocation of `#!sase/fix_just` in the SASE project.

Do not remove or alter the barrier in the GitHub Actions fixer's `build_prompt()`, because that source launches a normal
agent and contains no `#!` reference.

### 2. Turn the chezmoi inventory test into a policy regression

Replace the existing “all agent-backed chops have a barrier” invariant with an invariant that reflects prompt kind:

- Parse the Athena YAML rather than relying on line-oriented matching.
- Pin the same eight `agent:` chop names so inventory drift remains visible.
- Require every pinned prompt to contain its explicit standalone `#!` reference and zero `%w(runners=0)` occurrences.
- Rename the test file/function if needed so their names describe standalone-workflow exclusion rather than barrier
  presence.

Keep PyYAML in the development requirements because the configuration test should continue parsing the actual YAML
structure.

Update the executable `sase_fix_just` regression test to assert that the captured `sase run -d` prompt still contains
`#!sase/fix_just`, preserves the expected name/VCS/group directives, and contains no runner barrier.

Keep the GitHub Actions fixer regression asserting exactly one barrier, and additionally make the distinction explicit
by asserting its captured prompt has no `#!` standalone-workflow marker. Together, these tests cover both sides of the
policy for executable chops.

### 3. Preserve and clarify the SASE leaf-agent guarantees

Do not change the four bundled workflow prompt sources or remove their barriers. Extend their focused launcher tests to
assert that every captured barrier-bearing detached prompt has no `#!` reference, alongside the existing assertions
that:

- each ordinary root-agent prompt has exactly one runner barrier;
- directive extraction yields `wait_runners == 0` and strips the directive from model-visible text; and
- the split-file and refresh-docs named dependency chains remain intact.

This protects the intended wrapper/leaf boundary if a bundled workflow is later refactored to dispatch another
standalone workflow instead of launching an ordinary agent.

## Risks and Mitigations

- **Removing barriers from real leaf agents:** constrain removal to prompts containing the explicit `#!` marker and
  retain positive one-barrier assertions for the GitHub Actions fixer and bundled detached prompts.
- **Confusing inline xprompts with standalone workflows:** key the rule specifically on `#!`; ordinary `#name`
  references remain eligible for barriers.
- **Losing workflow launch directives or arguments:** make minimal prompt edits and assert the expected `#!` reference,
  name, VCS/group directives, and inventory names in focused tests.
- **Future inventory drift:** parse and pin all current Athena `agent:` chops, failing loudly if a standalone wrapper is
  added with a barrier or if the expected set changes.
- **Prompt-hash duplicate launches during rollout:** quiesce affected chops before applying the corrected prompt text.

## Out of Scope

- Rejecting `%w(runners=0)` plus `#!` globally in the directive parser or launch APIs.
- Automatically injecting or removing runner directives based on prompt contents.
- Changing standalone workflow syntax, flattening, execution, or HITL behavior.
- Changing the runner FIFO gate, thresholds, axe concurrency limits, schedules, or deduplication formats.
- Removing barriers from normal agent prompts merely because they contain inline `#name` xprompts.
- Launching chops as part of validation.
